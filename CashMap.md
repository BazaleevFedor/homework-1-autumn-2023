# Тестовый отчет проекта <a href="http://212.233.91.72/" target="_blank">Depeche</a> 

## 1. Регистрация
#### Успешные тест-кейсы
- При подсветке полей ввода данных красным цветом высвечивается текст, указывающий на необходимость корректного заполнения полей.
- При отсутствии подстветки полей красным и нажатии на кнопку регистрации происходит переход на новостную ленту нового пользователя.
- При нажатии на текст "У вас уже есть аккаунт? Войти" происходит переход на страничку авторизации.
- При повторной регистрации одного и того же пользователя выводится сообщение "Пользователь с таким email уже существует".
- При вводе цифры в поле "Имя" выводится сообщение "Имя должно содержать буквенные символы" и поле становится красным.
- При вводе цифры в поле "Фамилия" выводится сообщение "Фамилия должна содержать буквенные символы" и поле становится красным.
- При вводе электронной почты без символа "@" выводится сооющение "Некорректный адрес электронной почты" и поле становится красным.
- При вводе пароля короче 8 символов выводится сообщение "Пароль должен быть длиннее 8 символов" и поле становится красным.
- При несовпадении значений полей "Пароль" и "Повторите пароль" выводится сообщение "Введите пароль" и второе поле становится красным.
- При отсутствии текста в одном из полей на момент нажатия кнопки регистрации поле становится красным, появляется просьба его заполнить.
- При исправления ошибки заполнения поле перестает быть красным, пропадает сообщение об ошибке. 

#### Дефекты
- При переходе на страницу авторизации и возвращении на страницу регистрации пропадает подсветка ошибок, текст при этом остается в полях ввода.
- При переходе на страницу авторизации и возвращении на страницу регистрации сохраняется не последний введенный текст, а текст уже исправленной ошибки.

## 4. Навигация
#### Успешные тест-кейсы
- При наведении на кнопку сайдбара происходит затеменение соответствующей кнопки.
- При нажатии на кнопку сайдбара происходит переход на соответствующую страницу веб-сервиса, цвет текста кнопки становится фиолетовым.
- При смене страниц меняется урл.
- При открытии меню сообществ открывается страница сообществ с подменю "Мои сообщества".


#### Дефекты
- При нажатии на уже нажатую кнопку сайдбара происходит переход на ту же страницу с перерендерингом всех элементов.


## 5. Меню сообществ
#### Успешные тест-кейсы
- При нажатии на подменю в правом сайдбаре открывается определенный подраздел, выбраное меню выделяется фиолетовым цветом текста.
- При нажатии на кнопку "Создать сообщество" открывается модальное окно с полями "Название" и "Описание".
- При попытке создать сообщество с пустым именем высвечивается сообщение "Это поле не может быть пустым".
- При успешном создании сообщества открывается вкладка "Мои сообщества", создатель автоматически становится подписчиком созданого сообщества.
- При вводе названия сообщества в меню "Поиск сообществ" снизу высвечиваются сообщества, содержащие строку в названии.

#### Дефекты
- При смене раскладки клавиатуры или переходе на другую страничку браузера во время создания нового сообщества отменяется его создание.
- При создании сообщества с пустым названием высвечивается ошибка, но поле не выделяется красным цветом.

## 5. Страница сообщества

- При вводе пустой строки вместо названия в меню управления сообщестом высвечивается сообщение "Введите название" и поле становится красным.
- При переходе на страничку сообщества подписчиком в шапке отображается кнопка "Отписаться" и надпись "Вы подписаны".

## 6. Профиль пользователя
#### Успешные тест-кейсы
- При нажатии на кнопку "Редактировать", открываются настройки пользователя.
- При отсутствии на странице пользователя записей, выводится заглушка "Тут пока пусто"
- При открытии профиля подгружается информация, когда пользователь был в сети.
- При изменении фотографии в профиле, фотография меняется также в профиле, navbar'е и всех написанных комментариях и созданных записях.
- После редактирования персональных данных, при нажатии на кнопку "Сохранить", данные сохраняются.
- В случае некорректно введенных данных при редактирование профиля, соответствующие формы ввода подсвечиваются красным и выводится сообщение об ощибке.
- При изменении аватарки, обложка страницы меняет свой цвет, высчитывая главные цвета новой аватарки.

#### Дефекты
- У иконок редактирования и удаления комментария отсутствует поведение при наведении.
- При пустом тексте комментария кнопка отправки остается активной.

## 7. Новостная лента
#### Успешные тест-кейсы
- В ленте новостей отображаются записи друзей, собственные записи и записи сообществ, на которые подписан пользователь.
- При создании новой записи, она сразу отображается в летне.
- При написании комментария или добавления лайка, их количество в соответствующем счетчике увеличивается.
- При нажатии на кнопку "Удалить запись", она удаляется.
- При нажатии на кнопку "Редактировать", появляется возможность изменить свою запись.

#### Дефекты
- При обновлении страницы отрисовывается заглушка, а потом уже на ее месте появляются все записи.

## 8. Мессенджер
#### Успешные тест-кейсы
- Кнопка отправки сообщения остается неактивной до тех пор, пока текст сообщения пустой.
- При нажатии на иконку смайлика, открывается клавиатура смайликов и стикеров.
- При нажатии на иконту фотоаппарата, появляется возможность загрузить изображение.
- При нажатии на иконку назад, осуществляется переход на станицу со всеми чатами.
- При нажатии на имя собеседника в чате, осуществляется переход на его страницу.

#### Дефекты
- При отправке сообщения оно не осразу отображается на странице, а только после ее обновления.
- При нажатии на кнопку "Написать сообщение" появляется пустой диалог у обоих пользователей, даже если ничего не писать в сообщении.
- Нет опознавательного знака, что чат можно прокрутить вниз и там присутствует еще информация.
- При последнем многострочном сообщении карточки чата растягивается по вертикали и не происходит обрезка текста сообщения.

## 9. Друзья
#### Успешные тест-кейсы
- При отсутствии друзей у пользователя, в разделе "Друзья" будет выведена заглушка "У вас пока нет друзей".
- При отсутствии подписчиков у пользователя, в разделе "Подписчики" будет выведена заглушка "У вас пока нет подписчиков".
- При отсутствии подписок у пользователя, в разделе "Подписки" будет выведена заглушка "У вас пока нет подписок".
- При отсутствии друзей у пользователя, при переходе на страницу друзья, он будет перенаправлен на раздел "Поиск друзей".
- При нажатии на кнопку "Добавить в друзья" в разделе "Поиск друзей", данный пользователь перемещается в список подписок.
- При нажатии на кнопку "Добавить в друзья" в разделе "Подписчики", данный пользователь перемещается в список друзей.
- При нажатии на кнопку "Добавить в друзья" в разделе "Подписчики", данный пользователь перемещается в список друзей.
- При нажатии на кнопку "Написать сообщение" в разделе "Друзья", открывается диалог с выбранным пользователем.
- При нажатии на кнопку "Удалить из друзей" в разделе "Друзья", данный пользователь перемещается в список подписчиков.
- При нажатии на кнопку "Страница" в разделе "Друзья", открывается страница выбранного пользователя.
- При нажатии на кнопку "Отписаться" в разделе "Подписки", данный пользователь ужаляется из списка подписок.
- При выборе конкретного раздела страницу, открывается соответствующий раздел.

#### Дефекты
- При переходе на страницу друзей со страницы сообществ, если у пользователя пока нет друзей, открывается подраздел "Друзья" с заглушкой "У вас пока нет друзей" вместо подраздела "Поиск друзей".
- При нажатии на кнопку "Отклонить" в разделе "Подписчики", никаких действий не происходит.
- При нажатии на карточку пользователя, не осуществляется переход на его страницу. Переход осуществляется только при нажатии на имя пользователя или его аватарку.
